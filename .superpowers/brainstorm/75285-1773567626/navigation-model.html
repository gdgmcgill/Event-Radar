Club Dashboard Navigation Model

How should the club dashboard relate to the main app navigation?

A

Integrated with AppShell

Keep the main app's side nav and header. Add a secondary tab bar (Overview / Events / Members / Analytics / Settings) inside the content area. The dashboard feels like a "page within the app."

B

Dedicated Dashboard Layout

Replace AppShell entirely — no main nav or footer. Dashboard has its own minimal top bar with "Back to app" link and club-specific tabs. Like a separate admin panel.

C

Hybrid — Collapsed Sidebar

Keep the main AppShell sidebar but collapse it to icon-only mode. Club dashboard gets full content width with its own tab navigation. Hover the sidebar to expand it.

Recommendation: Option B

  • Club organizers spend long sessions in the dashboard — they need maximum screen real estate
  • Separate context signals "you're managing your club now" — reduces cognitive load
  • "Back to app" link keeps them oriented without eating space
  • No double nav problem — clean, purpose-built layout

Why not A or C?

  • A: Main sidebar wastes 200px on every page — organizers don't need Home/Events/Profile while managing their club
  • C: Collapsed sidebar still takes space and adds hover complexity on mobile
