**Section 2 — Backfill existing club creators to owner role**

```sql
UPDATE public.club_members
SET role = 'owner'
WHERE (user_id, club_id) IN (
  SELECT created_by, id FROM public.clubs

```
Place this BEFORE the CHECK constraint DO block in the file. Rationale: if any existing role values are unexpected, the UPDATE normalizes them. If all rows already have 'organizer', this changes the creator rows to 'owner' (which satisfies the constraint). Then the ADD CONSTRAINT validates cleanly.

Actually, to be safe: place the backfill FIRST (it changes 'organizer' to 'owner' for creators), then the CHECK constraint (which validates all remaining rows are either 'owner' or 'organizer'). The backfill is idempotent — running it again just sets rows that are already 'owner' to 'owner'.

```sql
CREATE OR REPLACE FUNCTION public.is_club_owner(p_club_id UUID)
RETURNS boolean
LANGUAGE sql
STABLE
SECURITY DEFINER
SET search_path = public
AS $$
  SELECT EXISTS (
    SELECT 1 FROM public.club_members
    WHERE club_id = p_club_id
      AND user_id = auth.uid()

  );
$$;
```
