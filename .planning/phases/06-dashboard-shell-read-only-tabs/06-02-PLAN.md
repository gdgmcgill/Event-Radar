**Step 1: Create `src/components/clubs/ClubEventsTab.tsx`**

This is a `"use client"` component that fetches and displays club events.

1. Define `ClubEventsTabProps` interface: `{ clubId: string }`.

2. Export `ClubEventsTab` function component with state:
   - `events: Event[]` (initially empty)
   - `loading: boolean` (initially true)
   - `error: string | null` (initially null)

3. `useEffect` on mount (dependency: `[clubId]`):
   - Fetch `GET /api/clubs/${clubId}/events`
   - On success: parse JSON, set `events` from `data.events ?? []`
   - On error: set `error` to "Failed to load events"
   - In finally block: set `loading` to false

4. Render structure:
   - **Header row** (flex justify-between items-center mb-6):
     - Left: `<h2>` "Events" heading
     - Right: `<Link href={/create-event?clubId=${clubId}}>` wrapping a `<Button>` with `<Plus>` icon and text "Create Event"

   - **Loading state**: Show a centered `<Loader2>` spinner with "Loading events..." text when `loading` is true

   - **Error state**: Show error message in destructive text with a "Try Again" button that reloads events (call the fetch function again, or use `window.location.reload()` — prefer re-fetching by extracting the fetch into a named function called from useEffect and from the retry button)

   - **Empty state**: When `events.length === 0` and not loading/error:
     - Centered layout with `CalendarX2` icon (from lucide-react), "No events yet" heading
     - Subtitle: "Create your first event for this club."
     - `<Link>` to `/create-event?clubId=${clubId}` with a Button

   - **Events list**: When events exist, render a vertical list (space-y-3):
     - Each event as a row/card (`div` with `p-4 rounded-lg border bg-card hover:bg-accent/50 transition-colors`):
       - **Left section**: Event title as a bold link to `/events/${event.id}`, event date formatted using `formatDate(event.event_date)` and time using `formatTime(event.event_time)` in muted text, location in muted text with `MapPin` icon
       - **Right section**: Status badge using `<Badge>`:
         - `"approved"` → variant "default" with green styling (className `bg-green-100 text-green-800 dark:bg-green-900 dark:text-green-200`)
         - `"pending"` → variant "secondary" with amber styling (className `bg-amber-100 text-amber-800 dark:bg-amber-900 dark:text-amber-200`)
         - `"rejected"` → variant "destructive"

5. Imports needed: `useState`, `useEffect` from "react"; `Link` from "next/link"; `Button` from `@/components/ui/button`; `Badge` from `@/components/ui/badge`; `Plus`, `Loader2`, `CalendarX2`, `MapPin` from "lucide-react"; `Event` type from `@/types`; `formatDate`, `formatTime` from `@/lib/utils`.
