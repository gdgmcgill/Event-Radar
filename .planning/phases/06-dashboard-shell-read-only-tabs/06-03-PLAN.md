<gap_analysis>
**Root cause:** `GET /api/clubs/[id]/events` (src/app/api/clubs/[id]/events/route.ts) returns raw Supabase rows where the date column is `start_date`. ClubEventsTab.tsx reads `event.event_date` and `event.event_time` — fields that only exist after `transformEventFromDB` (from src/lib/tagMapping.ts) is applied.

Every other event API route in the codebase applies `transformEventFromDB` before responding:
- `GET /api/events` — line 400: `rawEvents.map(event => transformEventFromDB(event as ...))`
- `GET /api/events/[id]` — uses `transformEventFromDB`
- `GET /api/events/popular` — uses `transformEventFromDB`
- `GET /api/events/happening-now` — uses `transformEventFromDB`
- `GET /api/recommendations` — uses `transformEventFromDB`
- `GET /api/users/saved-events` — uses `transformEventFromDB`

The clubs events route is the ONLY event endpoint missing this transformation.

**Runtime effect:** `formatDate(undefined)` returns "undefined". `formatTime(undefined)` returns "NaN:undefined AM". Every event in the Events tab has broken date/time display.

**Fix scope:** Single file change — add `import { transformEventFromDB } from "@/lib/tagMapping"` and `.map(transformEventFromDB)` before returning the response. No changes to ClubEventsTab.tsx needed (it already correctly reads Event type fields).
</gap_analysis>
