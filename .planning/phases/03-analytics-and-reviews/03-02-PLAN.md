   ```typescript
   export interface Review {
     id: string;
     user_id: string;
     event_id: string;
     rating: number;
     comment: string | null;
     created_at: string;
   }

   export interface ReviewAggregate {
     average_rating: number;
     total_reviews: number;
     distribution: { rating: number; count: number }[];
     comments: { rating: number; comment: string; created_at: string }[];
   }
   ```

2. **Create supabase/migrations/020_reviews_table.sql:**
   - Use idempotent pattern (IF NOT EXISTS / DO $$ blocks) matching project convention from Phase 1
   - Create `reviews` table with: id (UUID PK default gen_random_uuid()), user_id (UUID NOT NULL references auth.users ON DELETE CASCADE), event_id (UUID NOT NULL references events ON DELETE CASCADE), rating (INTEGER NOT NULL CHECK >= 1 AND <= 5), comment (TEXT nullable), created_at (TIMESTAMPTZ default now()), updated_at (TIMESTAMPTZ default now())
   - UNIQUE constraint on (user_id, event_id) to enforce one review per user per event
   - Enable RLS on reviews table
   - RLS policies: (a) users can INSERT their own reviews (auth.uid() = user_id), (b) users can SELECT their own reviews, (c) club organizers can SELECT reviews for their club's events (join events -> club_id -> club_members)
   - Indexes: idx_reviews_event_id on event_id, idx_reviews_user_event on (user_id, event_id)
   - **IMPORTANT:** This migration is a SQL file for reference. The actual table must be created via the Supabase SQL editor by the user. Add a comment at the top of the file explaining this.

3. **Create GET + POST /api/events/[id]/reviews/route.ts:**

   **POST handler (submit review):**
   - Auth check via supabase server client `auth.getUser()`. Return 401 if not authenticated.
   - Parse body: `{ rating: number, comment?: string }`
   - Validate rating is integer 1-5. Return 400 if invalid.
   - **Eligibility check 1 - Event has ended:** Query event's `start_date`, check `new Date(event.start_date) < new Date()`. Return 400 "Reviews can only be submitted after the event has ended" if not.
   - **Eligibility check 2 - User RSVP'd going:** Query `rsvps` where event_id + user_id + status='going'. Return 403 "Only attendees who RSVP'd going can review" if not found.
   - **Eligibility check 3 - No existing review:** Query `reviews` where event_id + user_id. Return 409 "You have already reviewed this event" if exists.
   - Insert review into `reviews` table. Return 201 with the review object.
   - Wrap in try/catch, return 500 on unexpected errors.

   **GET handler (aggregate reviews):**
   - Auth check. Return 401 if not authenticated.
   - Determine caller role: check if user is a club member for this event's club (via events -> club_id -> club_members). Set `isOrganizer = true` if so.
   - Query all reviews for this event_id.
   - Calculate: average_rating (rounded to 1 decimal), total_reviews, distribution (count per rating 1-5).
   - Comments: If isOrganizer, return comments array with { rating, comment, created_at } -- NO user_id or names (anonymized per REVW-02). If not organizer, return empty comments array.
   - Also check if the current user has already reviewed (for UI state): include `user_review: Review | null` in response.
   - Also include `can_review: boolean` -- true if user RSVP'd going AND event ended AND no existing review.
   - Return `{ aggregate: ReviewAggregate, user_review: Review | null, can_review: boolean }`.

4. **Add useEventReviews hook to src/hooks/useAnalytics.ts** (file already exists from Plan 01):
   - `useEventReviews(eventId: string)` -- fetches `/api/events/${eventId}/reviews`, returns `{ data, error, isLoading, mutate }`

5. **Run tests and fix until green:** After creating the route file, run the test stubs from Task 0. Fix any issues in the route handlers until all tests pass.
