2. **Create GET /api/events/[id]/analytics/route.ts:**
   - Auth check: get user via supabase server client `auth.getUser()`
   - Authorization: verify user is member of the event's club via `club_members` table (join event -> club_id -> club_members where user_id matches). Return 403 if not a member.
   - Query `event_popularity_scores` for view_count, click_count, save_count, unique_viewers (use `.maybeSingle()` -- may not exist yet for new events, default to 0s)
   - Query `rsvps` for this event, count by status (going vs interested), exclude cancelled
   - Query `saved_events` count for this event
   - Return `EventAnalytics` object. Use saves from `saved_events` count (more accurate than popularity_scores.save_count).
   - Handle empty data gracefully: return zeros, not errors.
   - Follow existing API error handling pattern: try/catch, NextResponse.json with appropriate status codes.

3. **Create GET /api/clubs/[id]/analytics/route.ts:**
   - Auth check + club membership verification (same pattern as event analytics)
   - **Follower growth:** Query `club_followers` for this club, select `created_at`, order ascending. Use date-fns `eachDayOfInterval` and `format` to bucket by day over the last 30 days. Return cumulative count array: `{ date: "Mar 1", count: 5 }[]`. Handle empty followers (return empty array).
   - **Total attendees:** Query `rsvps` joined with `events` where `events.club_id` matches AND `rsvps.status = 'going'`. Count total. Use a single query: `supabase.from("rsvps").select("id, events!inner(club_id)").eq("events.club_id", clubId).eq("status", "going")` -- count the result length.
   - **Popular tags:** Query all `events` for this club, collect all tags arrays, flatten, count occurrences, return top 5 sorted descending: `{ tag: string, count: number }[]`.
   - **Events analytics:** For each event belonging to this club, fetch its analytics (popularity scores + RSVP counts) in bulk queries (NOT N+1). Query all `event_popularity_scores` where event_id in club's event IDs. Query all `rsvps` grouped by event_id. Combine into `EventAnalytics[]` sorted by event_date descending.
   - Return `ClubAnalytics` object.

4. **Create src/hooks/useAnalytics.ts:**

   - Use same fetcher pattern as useClubs.ts (simple fetch + json parse, throw on !ok)
