**1. Modify `src/app/api/clubs/[id]/events/route.ts`:**
- Add auth check at the start: call `supabase.auth.getUser()`. If authenticated, check `club_members` table for membership with this club.
- If user IS a member: remove the `.eq("status", "approved")` filter so all events are returned. Also add RSVP count aggregation.
- If user is NOT a member (or not authenticated): keep existing behavior (approved only, no RSVP counts).
- For RSVP counts: after fetching events, collect all event IDs, batch query `rsvps` table with `.in("event_id", eventIds)`, group counts client-side by event_id and status (going/interested). Attach `rsvp_counts: { going: number, interested: number }` to each event in the response.
- Use `start_date` for ordering (not `event_date`) and sort descending (most recent first).
- Return `{ events, isOrganizer: true/false }` so the client knows which view it got.

**2. Modify `src/app/api/events/create/route.ts`:**
- After the event is successfully inserted with `status === "approved"` and `body.club_id` exists, add notification fanout:
  - Fetch club name: `supabase.from("clubs").select("name").eq("id", body.club_id).single()`
  - Fetch followers: `supabase.from("club_followers").select("user_id").eq("club_id", body.club_id)`
  - If followers exist, batch insert notifications: `supabase.from("notifications").insert(followers.map(f => ({ user_id: f.user_id, type: "new_event", title: "New Event", message: "${clubName} published: ${title}", event_id: data.id, read: false })))`
  - Use fire-and-forget pattern: do NOT await the notification insert. Wrap in `.then(() => {}).catch(err => console.error("Notification fanout error:", err))` so it doesn't block the response.
- The response already returns `{ event, message }` -- no change needed there.

**3. Add `useClubEventsManagement` to `src/hooks/useClubs.ts`:**
- Add a new hook that fetches `/api/clubs/${clubId}/events` (same endpoint, but now returns all statuses for organizers).
- Return type should include `events`, `isOrganizer`, `isLoading`, `error`, `mutate`.
- This is separate from `useClubEvents` to keep the public-facing hook unchanged.
- Follow the existing SWR pattern with the same `fetcher`.
