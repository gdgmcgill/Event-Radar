**1. Extend `src/components/events/CreateEventForm.tsx` to support edit and duplicate modes:**

Add new props to `CreateEventFormProps`:
```typescript
interface CreateEventFormProps {
  clubId?: string;
  onSuccess?: () => void;
  initialData?: {
    title: string;
    description: string;
    start_date?: string;
    location: string;
    tags: EventTag[];
    image_url?: string | null;
    category?: string | null;
  };
  eventId?: string;       // If set, form is in edit mode (PATCH)
  mode?: "create" | "edit" | "duplicate";
}
```

- Default `mode` to `"create"` if not provided.
- In the component initialization, if `initialData` is provided, populate the form state from it:
  - `title` = initialData.title
  - `description` = initialData.description
  - `location` = initialData.location
  - `tags` = initialData.tags
  - For edit mode: parse `initialData.start_date` to extract date (YYYY-MM-DD) and time (HH:MM) parts to populate the date/time fields.
  - For duplicate mode: leave date and time EMPTY (user must pick new date).
  - If `initialData.image_url` exists, set `imagePreview` to that URL (but `imageFile` stays null since it's an existing URL).
- In `handleSubmit`:
  - If `mode === "edit"` and `eventId` is set: send `PATCH /api/events/${eventId}` with the updated fields instead of `POST /api/events/create`. Only send fields that changed from initialData to reduce payload. Include `image_url` if a new image was uploaded, otherwise keep the existing one.
  - If `mode === "duplicate"` or `mode === "create"`: keep existing POST behavior.
- In the success screen:
  - Edit mode: show "Event Updated!" with "Your changes have been saved."
  - Duplicate/create mode: keep the dynamic message from Plan 02-01.
- Change submit button text based on mode: "Create Event" (create/duplicate), "Save Changes" (edit).
- Change form title context: the `CreateEventModal` title will handle this externally, no change needed in the form itself.

**2. Update `src/components/clubs/ClubEventsTab.tsx` to add Edit and Duplicate buttons:**

- Add state for edit/duplicate modals: `editingEvent` (Event | null), `duplicatingEvent` (Event | null).

  - Edit button (Pencil icon from lucide-react): sets `editingEvent` to that event.
  - Duplicate button (Copy icon from lucide-react): sets `duplicatingEvent` to that event.
- Use shadcn `Button` with `variant="ghost"` and `size="icon"` for the action buttons.

  - `initialData` built from the editingEvent object

  - `onSuccess` closes modal and calls `mutate()` to refresh the list
  - Dialog title: "Duplicate Event"
